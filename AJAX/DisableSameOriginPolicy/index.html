AJAX Research - Disable Same Origin Policy

The same origin policy is enforced by browers to prevent requests from a web site on one domain from succeeding when making requests to web services running on another domain. This is great in a production environment but often in development it is desirable to have services running on a different server than a web site.

Some browsers allow disabling the same origin policy. For example, Chrome can be started using --disable-web-security and then allow requests to succeed. This test page can be used to verify that the same origin policy has been disabled.

Instructions

Start Chrome at the command line and supply the --disable-web-security option. Navigate to this page to verify that the same origin policy has been disabled.
